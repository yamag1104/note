# note記事制作ワークフロー 完全版
## リサーチ → コンセプト → 目次 → 本文 → テキスト化 → 有料設定 → SNS告知

---

## 全体方針（らっこ式60〜70点ルール）

> **完璧を求めない。60〜70点の記事を1ヶ月テスト発信して、市場の反応を見てから改善する。**
> 最初から100点を目指すと発信できない。まず出して、反応をデータにする。

---

## 全体フロー

```
STEP1  市場調査リサーチ（ジャンル選定3要素で評価）
         ↓
STEP2  コンセプト整理（三大欲求×HARMの法則）
         ↓
STEP3  目次作成
         ↓
STEP4  本文執筆（はじめに・第1〜6章・おわりに）
         ↓
STEP5  Wordファイル出力
         ↓
STEP6  有料設定ライン・価格設定
         ↓
STEP7  SNS告知文作成・投稿（X→note順序を守る）
```

## STEP1｜市場調査リサーチ

### らっこ式：ジャンル選定3要素（リサーチ前に必ず評価）

> **売れるジャンルの条件：市場の大きさ × 悩みの深さ × 悩みの緊急度**

| 要素 | 評価基準 | 合格ライン |
|------|---------|----------|
| 市場の大きさ | そのジャンルに悩む人が多いか | Xで検索して投稿数・フォロワー数を確認 |
| 悩みの深さ | 解決したくて夜も眠れないレベルか | 表層的な不便ではなく「本音の苦しみ」か |
| 悩みの緊急度 | 今すぐ解決したいか・後回しにできないか | 「いつかやろう」ではなく「今すぐ知りたい」 |

**らっこ式ジャンル別収益規模の目安**

| ジャンル | 月間売上規模 | 価格帯 | 特徴 |
|---------|------------|--------|------|
| お金・副業 | 大 | 1,000〜5,000円 | 緊急度高、競合多い |
| 健康・ダイエット | 大 | 500〜3,000円 | 需要安定、悩み深い |
| 恋愛・結婚 | 中〜大 | 1,000〜3,000円 | 感情購買、拡散しやすい |
| 子育て・教育 | 中 | 500〜2,000円 | ママ層の購買力高い |
| キャリア・転職 | 中 | 1,000〜3,000円 | 決断期に集中購買 |
| 趣味・ライフスタイル | 小〜中 | 300〜1,500円 | 競合少、ファン化しやすい |

### らっこ式：HARMの法則（インサイト分析の軸）

ターゲットの悩みをこの4軸で分類して深掘りする。

| 軸 | 意味 | 悩みの例 |
|----|------|---------|
| **H**ealth（健康） | 身体・精神・美容の悩み | 疲れが取れない／老けた気がする |
| **A**mbition（野心） | 成長・スキル・将来への不安 | このままでいいのか／AIに仕事を奪われる |
| **R**elation（人間関係） | 家族・職場・恋愛の悩み | 子どもと話が噛み合わない／夫婦の温度差 |
| **M**oney（お金） | 収入・節約・資産形成 | 収入が増えない／老後が不安 |

> **ポイント**：ターゲットの「本音の悩み」はたいてい複数の軸にまたがる。
> 例：「AI時代の主婦の悩み」= A（時代に取り残される不安）× R（子どもに何も教えられない）× M（収入がない）

### 確認する4つの条件（必ず順番に聞く）

| # | 質問 | 選択肢例 |
|---|------|---------|
| ① | 調査テーマ | AI副業／思考法／副業術 など |
| ② | ターゲット層 | 主婦／20代会社員／フリーランス／学生 など |
| ③ | 販売形態 | eBook・ノウハウ記事／テンプレート・ツール／オンライン講座 |
| ④ | 重点項目 | インサイト深掘り／競合分析重視／ブルーオーシャン発見／パワーワード重視 |

### 出力する4項目

```
1. ターゲットの深い悩み（インサイト）× 5つ
   └ HARMの法則で分類して「夜も眠れない本音の動機」を抽出

2. 競合コンテンツの分析 × 3件
   └ タイトル・価格帯・訴求ポイント・弱点

3. 市場の空白（ブルーオーシャン）× 3つ
   └ 競合がまだ触れていないニッチな需要

4. 売れるパワーワード × 10個
   └ ターゲットの感情を揺さぶる単語
```

## STEP2｜コンセプト整理

### らっこ式：三大欲求（コンセプトの根本に置く）

> **人が情報にお金を払う動機は3つだけ。コンセプトをこの欲求に接続する。**

| 欲求 | 意味 | 購買トリガー |
|------|------|------------|
| **繁殖欲求** | 子どもを守りたい・家族を守りたい | 「子どもの未来を守る」「家族が安心できる」 |
| **承認欲求** | 認められたい・尊敬されたい・自己成長 | 「周りに差をつける」「一歩先を行く親になる」 |
| **金銭欲求** | お金を増やしたい・損したくない | 「月1万円の副収入」「AI時代に稼ぐ方法」 |

> **使い方**：タイトルとキャッチコピーに最低1つの欲求を必ず組み込む。
> 複数の欲求を組み合わせると購買力が上がる（例：繁殖×承認＝「子どもに尊敬されるお母さんになる」）

### 入力する情報

```
・ターゲット：{ターゲット}
・切り口：市場の空白になっているところ
・パワーワード：{リサーチで選んだパワーワード2〜3個}
・接続する欲求：{繁殖／承認／金銭 から選ぶ}
```

### 出力する4要素

| 要素 | 内容 |
|------|------|
| タイトル案 | 5本（パワーワードを2つ以上＋三大欲求を1つ以上含める） |
| 誰のどんな悩みを解決するか | インサイトから抽出した「本音の問題」 |
| どうなれるか | 記事を読んだ後の読者の変化 |
| 独自の売り | 競合にない切り口・市場の空白 |

### タイトル作成の公式

```
【ターゲット】 + 【パワーワード①】 + 【パワーワード②】
例：「子どもに教えられるお母さんになる　AI時代の家族を守る思考法」
```

## STEP3｜目次作成

### Xでニーズテスト（目次・執筆前の最重要チェック）

> **売れる記事の種は、あなたのXのポストの中に存在する。**
> 記事を書く前に、そのテーマについて需要があるのかをXのポストでテストすること。

**やり方：**
```
① 普段の投稿の中で、いつもより「いいね・リポスト・リプライ」が多かったテーマを探す
② Xで「○○というテーマでnoteの記事を書こうと思っているんですが、
   需要はありますか？いいねやリブで教えてほしいです！」とポストして反応を確認する
③ 反応が良ければ記事化→ 反応が薄ければ別のテーマに転換する
```

> **この鉄則を、絶対に忘れないでください。**
> どれだけ素晴らしい内容の記事を書きあげても、そこに需要がなければ1円にもなりません。

### 6章構成ルール

| 章 | テーマ | 役割 | 狙い |
|----|--------|------|------|
| 第1章 | 問題提起 | 読者の現状を言語化 | 「私のことだ」と思わせる |
| 第2章 | 憧れ | 理想の姿を見せる | 「こうなりたい」と思わせる |
| 第3章 | 解決策 | 具体的な方法を提示 | 「これならできる」と思わせる |
| 第4章 | 証拠 | データ・実例で裏付け | 「本当に効果がある」と信じさせる |
| 第5章 | 提案 | ステップ別アクション | 「今日からできる」と感じさせる |
| 第6章 | 行動 | 最初の一歩を促す | 「やってみよう」と動かす |

各章に「キャッチーな見出し」＋「2〜3個のサブ見出し」をつける。

### 目次チェックリスト（売上を倍増させる目次の条件）

> **目次こそが、読者が購入を判断する上で非常に重要な役割を果たす。**
> 目次を読んだだけで「この記事を読めば、自分の悩みが解決できそうだ」「これは面白そうだ」とワクワクする構成になっているか徹底的に追求してください。

- [ ] 同ジャンルの書籍やnoteの記事の目次に目を通したか
- [ ] 同ジャンルの書籍やnoteの記事の目次で参考にできそうな内容は盛り込んでいるか
- [ ] 同ジャンルの書籍やnoteの記事の目次で抜けていた内容を盛り込めているか
- [ ] 目次を比べたとき、同ジャンルの売れている記事よりもあなたの記事のほうが豪華な内容か
- [ ] あなたの目次に「世間の常識とは逆の真実」もしくは「裏技感」のある内容が盛り込まれた章があるか
- [ ] 目次を見ただけで、欲しいと思えるような魅力的な目次になっているか

> **売上を倍増させる目次テクニック：**「世間の常識とは逆の真実」や「裏技感」のある内容を目次に入れる。
> 例：ダイエットジャンルで「運動よりも食事が9割」というテーマを立てる（常識を覆す新しい発見）

## STEP4｜本文執筆

### 執筆の黄金ルール（らっこ式）

> **いきなり無料エリアから書き始めるのではなく、まずはコンテンツの核となる有料エリアから書き始めること。**
> 有料エリアが完成したら無料エリアを書く。有料エリアの質が高ければ口コミで次の記事も売れる。

```
執筆順序：
① 有料エリア（第3〜6章）を書く  ←「購入した人を絶対ガッカリさせない」内容を先に完成させる
② リサーチを深める
③ 無料エリア（はじめに・第1〜2章）を書く
④ タイトル・冒頭10行を磨く
```

### リサーチ方法

```
・書籍    ：同ジャンルで評価の高い本を最低5冊読む（Amazonで目次を確認→試し読み）
・note    ：同ジャンルで売れている有料記事を最低1冊は購入して読む
・英語圏  ：日本より半年〜1年早くトレンドが来る英語圏のYouTube・ネット記事を活用
           （英語が苦手ならAIで翻訳すると海外の質の高い情報を取り込める）
・論文    ：説得力・信頼性を高めたい場合は論文も参考にする（AIで要約可能）
・AI      ：NotebookLMに書籍・論文・URLをアップロードして専用リサーチアシスタントにする
```

> **情報をまとめること自体に価値がある（キュレーション）。**
> 読者の時給が1,500円なら10時間のリサーチ＝15,000円分のコストを代わりに払っているのがあなた。

### 執筆ルール

```
・文字数    ：2,000文字程度／章（同ジャンルの売れている記事より多い文字数を目指す）
・トーン    ：親しみやすく、少し熱量のある口調
・構成      ：PREP法（結論→理由→具体例→結論）
・具体例    ：失敗談・成功例を必ず1つずつ入れる
・読者視点  ：「あなたは〇〇ではありませんか？」で引き込む
```

### PREP法の構造

```
【結論】  最初に「何を伝えたいか」をはっきり言う
   ↓
【理由】  「なぜそう言えるのか」根拠を説明する
   ↓
【具体例】 失敗談・成功例・データで実感させる
   ↓
【結論】  最初の結論を繰り返し、次章へ誘導する
```

### 執筆順序

```
① はじめに（フック・本書の目的）
② 第1章〜第6章（各2,000文字）
③ おわりに（復習・継続の呼びかけ・最後の一押し）
```

### 各章の書き出しパターン

| 章 | 冒頭パターン |
|----|------------|
| はじめに | 具体的な場面描写から始める（「ある夜のことです」） |
| 第1章 | 「こんな場面に心当たりはありませんか？」 |
| 第2章 | 「あなたの周りに、こんな人はいませんか？」 |
| 第3章 | 「〇〇と聞いて、こう思いませんでしたか？」 |
| 第4章 | 「でも、これって本当に効果があるの？」 |
| 第5章 | 「わかった。でも具体的に何から始めればいいの？」 |
| 第6章 | 「ここまで読んでくれて、ありがとうございます」 |
| おわりに | 読み始めたときの読者の気持ちを代弁する |

### らっこ式・無料エリアの黄金テンプレート（目安文字数3,000〜4,000字）

> **有料記事で一番大切なのは無料エリア。どんなに有料エリアが素晴らしくても、購入するかどうか読者が判断するのはあくまでも無料エリアで決まる。**
> 「読むことで買わない理由がなくなり、欲しくてたまらなくなる」状態まで読者をもっていくのが目標。

| # | 項目 | 目安文字数 | 狙い |
|---|------|-----------|------|
| ①【最重要】 | 冒頭の興味付け | 〜100字 | 読者の目を止め、「自分に関係がある」と思わせる |
| ② | 誰のどんな悩みを解決する記事か | 300〜400字 | ターゲットと提供価値を明確にし、「自分ごと化」させる |
| ③ | 読まないデメリット＆読んだあとの未来 | 300〜400字 | 行動しない悲惨な感情と、理想の未来を提示して感情を揺さぶる |
| ④ | 自己紹介 | 400〜500字 | なぜあなたがこれを語る資格があるのか、権威性を示し信頼させる |
| ⑤ | 執筆のきっかけ | 500〜1,000字 | ストーリーで共感を呼び、あなたから買いたいという理由を与えファン化させる |
| ⑥ | 目次 | — | 有料エリアへの期待感を最大化させる |
| ⑦ | 口コミ | 300〜400字 | 第三者の声で客観的な信頼性を担保する（モニターのスクリーンショット掲載） |
| ⑧ | 特典の紹介 | 300〜400字 | 豪華感を演出し、記事の価値を高める |
| ⑨ | オススメする人・しない人 | 300〜400字 | 読者をあえて絞り込むことで、「自分にこそ必要だ」と強く思わせる |
| ⑩ | 注意事項 | — | 「○部売れたら値上げします」など限定性を演出し希少価値を高める |
| ⑪ | 購入への心理的ハードルを下げる | 200〜300字 | 価格の妥当性を示し、お得感を演出する |
| ⑫ | 有料エリアへの寸止め | 100〜200字 | 最も気になるところで「続く」とし、購入ボタンを押させる最後のひと押し |

### タイトル・コピーライティング術

> **noteの記事は、第一印象がすべて。「タイトル」と「冒頭の10行」が非常に重要。**
> 何度も書き直し、複数の案の中から最高のものを厳選してください。

**売れるタイトルを作るテクニック：**

| 要素 | 使い方 |
|------|--------|
| **数字** | 「たった24時間で3,771万円のnote記事を販売した」「たった1週間で素敵な恋人ができた」など、数字でリアリティを加える |
| **キャッチーな言葉** | 読者の感情を揺さぶり、強く惹きつける言葉をタイトルに入れる |
| **キラーフレーズ** | 「知らないと損」「必須」「悪用厳禁」「神速」「完全攻略」など、少し大げさな表現も効果的 |
| **エモーショナルフレーズ** | 「思わず涙した」「心が軽くなる」など、読者が得られる感情を先に提示する |
| **フックフレーズ** | 「なぜ、○○はなんなのか？」「〜の正体」「秘密を公開」など答えを知りたくなるフック |
| **裏技感** | 「他の人が知らない方法でショートカットできる特別な情報」という印象を与える |
| **限定性** | 「プロだけが知る」「トップ1%の秘密」「元○○だから話せる」など情報の希少性をアピール |
| **常識を覆す** | 「実は○○は間違いだった」「○○はするな」など広く信じられている通説に一石を投じる |
| **簡単な解決策** | 「たった1つのコツ」「意外な近道」「これだけでOK」など解決の容易さを示唆する |

**「冒頭の10行」で惹きつけるテクニック：**

| パターン | 説明 |
|---------|------|
| **衝撃的な事実** | 「実は、あなたの努力が報われないのには明確な理由があります」→知的好奇心を刺激。書き手への権威性・専門性を感じさせる効果も |
| **読者の悩みを代弁** | 「もう何から手をつければいいかわからない、そう感じていませんか？」→「これはまさに私のことだ」という強烈な当事者意識をもたせる |
| **常識を覆す問いかけ** | 「もし、あなたが信じている○○が、実は間違っているとしたら？」→答えを知りたいという欲求が続きを読み進めさせる |

### 記事仕上げ確認リスト（書きあがったら最低1日置いて冷静な目で確認）

**記事の確認リスト：**
- [ ] 誤字脱字はないか
- [ ] 日本語として不自然な点はないか
- [ ] 読みにくい箇所はないか
- [ ] 音読して違和感がないか
- [ ] **スマホで見たとき、文字が詰まりすぎていないか**（改行をバラバラに入れ、画面が文字で埋め尽くされないよう注意）
- [ ] **太字部分だけを拾い読みしても、内容の骨子が理解できるか**

> **スマホでの読みやすさ最重要。** 読者はスマホで記事を見ていると想定し、適度な改行を入れる。
> 改行を統一せずバラバラで入れ、画面が文字で埋め尽くされないように。

**見出し画像（サムネイル）作成チェック：**
- [ ] Canvaで作成（noteの推奨サイズ：**1280×670px**）

## STEP5｜Wordファイル出力

### 実行コマンド

```bash
cd C:\Users\owner\dev\note
node create_article.mjs
```

### ファイル構成

```
C:\Users\owner\dev\note\
├── create_article.mjs     ← Word生成スクリプト
├── package.json
├── node_modules\
│   └── docx\
└── {記事タイトル}.docx    ← 完成ファイル
```

### スクリプトの基本構造

```javascript
import { Document, Packer, Paragraph, TextRun, HeadingLevel, AlignmentType } from "docx";
import { writeFileSync } from "fs";

const doc = new Document({
  styles: {
    default: {
      document: { run: { font: "游明朝", size: 24 } },
    },
  },
  sections: [{
    children: [
      // タイトル
      new Paragraph({
        text: "{タイトル}",
        heading: HeadingLevel.TITLE,
        alignment: AlignmentType.CENTER,
      }),
      // 見出し
      new Paragraph({ text: "第1章｜...", heading: HeadingLevel.HEADING_1 }),
      // 小見出し
      new Paragraph({ text: "結論：...", heading: HeadingLevel.HEADING_2 }),
      // 本文
      new Paragraph({ text: "本文テキスト..." }),
      // 強調テキスト
      new Paragraph({
        children: [new TextRun({ text: "強調したい文章", bold: true, size: 28 })],
        alignment: AlignmentType.CENTER,
      }),
    ],
  }],
});

const buffer = await Packer.toBuffer(doc);
writeFileSync("C:/Users/owner/dev/note/{ファイル名}.docx", buffer);
console.log("完成しました！");
```

## STEP6｜有料設定ライン・価格設定・販売テクニック

### らっこ式：売上が劇的に変わる9つの販売テクニック

> **記事を発売する"前"の段階で、これらのテクニックを無料エリアや告知ポストに盛り込む。**

| # | テクニック | 心理原則 | 使い方 |
|---|-----------|---------|--------|
| 1 | **引用特典をつける** | 社会的証明＋返報性 | 「感想をXでポストしてくれた方に〇〇をプレゼント」と記事末尾に記載 |
| 2 | **「限定」要素を付加する** | 希少性の原理 | 「100部限定」「10日間限定販売」など |
| 3 | **あえて「禁止」してみる** | カリギュラ効果 | 「本気で変わりたい人以外、絶対に買わないでください」 |
| 4 | **デメリットも訴求する** | 両面提示の法則 | 「このノウハウを実践するには最低でも1日1時間必要です」→信頼が上がる |
| 5 | **読者の認識を変化させる** | リフレーミング | 「1年以上かけて得た知識が、たった1回の飲み会代で手に入ります」 |
| 6 | **アフターフォローをつける** | 顧客満足＋返報性 | 購入者限定コミュニティ・期間限定質問回答（個別相談は工数がかかるので注意） |
| 7 | **「早く買わないと損」と思わせる** | 損失回避バイアス | 「先着100部限定で、通常2,980円のところ980円で販売します」 |
| 8 | **ベネフィットを伝える** | 未来の変化への訴求 | メリット（商品の特徴）ではなくベネフィット（読んだ後の生活がどう変わるか）を伝える |
| 9 | **無料モニターを募集する** | 社会的証明の先取り | 「5名限定で無料モニター募集！リポスト＆いいねで応募」→口コミを事前に集める |

> **大原則：価格以上の価値を必ず提供すること。テクニックはあくまで後押し。商品品質が最優先。**

### 発売前の口コミ収集（売上を加速させる）

> **記事を公開する前に、交流のあるフォロワーやモニター企画で集まってくれた方に先行して読んでもらい、感想をもらう。**
> そのロコミのスクリーンショットを無料エリアに掲載する。信頼性が格段に高まり「他の人がこんなに評価しているんだから、この記事は良いものに違いない」と感じた読者を購入に繋げやすくなる。

### 限定特典（売上を最大化させる）

> **記事を購入し感想をポストしてくれた方だけに渡す特典を準備すると、口コミが集まりやすくなる。**
> 「特典だけでも、価格以上の価値がある」と思ってもらえれば、あなたの記事が売れる可能性は飛躍的に高まる。

**特典の種類（最低3〜5個を用意）：**
- **関連テーマを深掘りした限定記事**：本編では書ききれなかった内容や特定のトピックをさらに詳しく解説した記事
- **チェックリスト**：記事で学べる内容を具体的な「やることリスト」に落とし込んだもの（Googleスプレッドシート・Wordで作成）
- **AIプロンプト集・テンプレート**：すぐに使えるもの

### noteのセール機能の使い分け

| 目的 | 使う機能 |
|------|---------|

### らっこ式：Xアカウント設計5ステップ（設計で売上の9割が決まる）

**ステップ1：市場・ジャンルを決定する**
- 「儲かる市場」と「自分が継続できる市場」の両立を探す

**ステップ3：コンセプト・キャラクターを決定する**
- 目指すのは「有料記事が売れるアカウントコンセプト・キャラクター」

- 例：先生・師匠・先輩の立場で発信する
- 自分の好きなことではなく、「何を専門にして発信するのか」を明確に絞る

**ステップ4：Xの全体像を理解する**
- ポスト → 日頃から反応してくれる人のTLに表示 → ユーザーがアクション → おすすめTLで拡散

**ステップ5：名前・アイコン・ヘッダー・プロフィールを整える**

| 要素 | ポイント |

| アイコン | 記憶に残るデザイン・先生や先輩のような「格上感」・ ゴナラなど5,000円〜で依頼可 |
| ヘッダー | 優先度低め・何について発信しているかを一言で |
| プロフィール | 110字前後・誰の悩みを解決するか・具体的な実績・現在の自分・常に改善を疑う |

**プロフィール文の構成：**

### 絶対にやってはいけないX運用10選

1. **相互フォローで伸ばす**（中身のないフォロワー数、エンゲージメントゼロ）
2. **定型文のようなリプまわり**（「すごいですね！」は逆効果。1日10人を目安に具体的なリプ）
3. **過度な「いいね」**（1度に4〜5人、1日合計30〜40回程度に留める）
4. **「いいね」をもらっても返さない**（ギブ＆テイク。返すことで次回も反応してもらえる）
5. **プロフィール画面の整理不足**（引用・リポストばかりで肝心の自分の発信が埋もれる）
6. **同じジャンルの発信者との交流不足**（横の繋がりが拡散の鍵）
7. **発信ジャンルの一貫性不足**（「この人は○○の人だ」と認識させることが非常に重要）
8. **価値提供への意識不足**（全体の7〜8割は「価値提供」にフォーカスする）
9. **ポストが具体的でない**（「今日からできそう」「知らなかった！」と感じさせるレベルまで落とし込む）
10. **リサーチ不足**（XもnoteもリサーチがX割。伸びているアカウントを常に観察する）

## AI活用法｜最短・最速で質の高い記事を作る

### AIとの正しい付き合い方

> **AIは「24時間文句も言わずに働いてくれる、超優秀なアシスタント」。タスクを丸投げするのではなく、二人三脚でタスクを進める。**
> AIを使いこなすコツは「習うより慣れろ」。使えば使うほど、どんなプロンプトを書けば望む答えが返ってくるのかわかるようになる。

**AIに最高の回答を出してもらうコツ：**
```
① 1回の指示で完璧な答えを求めない（AIと対話しながら成果物を改良していく）
② タスクを細かく分割して一つひとつクリアしていく
   悪い例：「noteの記事を書いて！」
   良い例：まずは構成案を考えて→次に導入部分を魅力的に書いて→本編を書いて→最後にまとめの文章を作って
③ 省略せず、具体的に細かく、すべてを伝える
④ 最後にこの一文を添えると精度アップ：
   「より良い出力をするために、私に問いたほうがいいことがあれば質問してください」
```

### AIを使った記事作成の流れ

> **いきなり記事を書くよりも、AIで「たたき台」を作り、そこから内容をブラッシュアップしていくほうが圧倒的に簡単に記事を作成できる。これが最も効率的で労力の少ない執筆方法。**
> AIで作った記事のほうが購入率が高い場合もある（AI執筆3.2% vs 外注ライター2.2%という実績も）。

```
① らっこ式・noteの記事を一撃で生成するGPTs を使う
   → 質問に答えるだけで記事のたたき台を自動生成
② 目次（構成）は必ず自分の目で細かく確認する
   抜けている内容は「○○の内容を○章に加えて」と指示して修正
③ 各章は「次の構成に従って、第○章を書いてください。[目次を貼り付け]」と章ごとに指示する
   （章ごとに出力させると文章の質が上がる）
④ 出来あがった章はコピーしてメモ帳に貼り付けておく
⑤ 書きあがったら誤字・日本語チェックもAIに依頼する
```

**AI文章チェック用プロンプト（コピペ用）：**
```
# 役割: note専門の校正・編集者

あなたは10年以上の経験を持つnoteの記事専門の編集者です。

以下の【ユーザーの文章】を、上記の役割と背景情報に基づいて校正・編集してください。

【優先度：高】
1. 誤字・脱字の修正
2. 文法的な誤り（てにをは、ら抜き言葉、主述のねじれ等）

3. 分かりやすさの向上（専門用語の言い換え、難解な表現の平易化）
```

## 長期戦略｜noteを「自動販売機」に変える

### 過去の記事を売れ続けさせる裏ワザ（8章）

> **一度質の高い有料記事を作れば、24時間365日働き続ける「自動販売機」になる。**
> 普遍的な悩みを解決する記事は、放っておいてもある程度売れ続ける。

**定期的な情報アップデートで価値を高める**
- 新しい情報・読者からの質問への回答を追記する
- 記事の冒頭に「○月○日、○○について追記しました！」と更新履歴を記載
- 更新したらXでポストして宣伝する → 新しい読者が記事を手に取るきっかけになる

### 挫折せず発信を「続ける」技術（8章）

> **継続できないのは意志が弱いからではなく、継続するための「技術」を知らないだけ。**

| 技術 | 内容 |
|------|------|
| **習慣化の設計** | ベイビーステップ：最初は「毎日Xのアプリを開くだけ」など脳への負荷を極限まで下げる |
| **今日やる宣言** | 仲間とのチャットグループで「今日やること」を宣言し、実行したら報告する |
| **完璧主義から卒業** | 一度できなかった日があっても「失敗は計画のうち」と捉えて翌日から淡々と再開 |
| **挫折耐性（セルフコンパッション）** | 失敗したとき「自分自身」と「出来事」を切り離して考える。自分を責めない |
| **仲間を作る** | 同じようにnoteで稼ごうとしている仲間を見つけて交流する |

### さらに売上を伸ばす上級戦略（8章）

```
新作記事を作り続ける
  → ひとつのヒットが次のヒットを生む「ドミノ倒し」現象
  → ファンになった読者が過去記事も遡って購入してくれる

有料マガジン機能の活用
  → 複数の有料記事をまとめてお得に販売
  → セット購入を促す

イベントセール
  → お正月・クリスマスなど季節のイベントにタイムセール（20%値下げ）
  → 期間限定販売（3〜5日間）で強力な希少性を演出（多用は厳禁）

価格のテスト（上級者向け）
  → いくつか記事を書いた後、価格を変えてどの価格が一番売れるか検証する
```

> **最低でも半年は続けること。**
> SNSやnoteは最初の数ヶ月は低空飛行が続くが、ある日を境に一気に伸びることがほとんど。
> その「ブレイクスルーポイント」まで諦めずに走り続けてください。
