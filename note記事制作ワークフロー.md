# note記事制作ワークフロー 完全版
## リサーチ → コンセプト → 目次 → 本文 → テキスト化 → 有料設定 → SNS告知

---

## 全体方針（らっこ式60〜70点ルール）

> **完璧を求めない。60〜70点の記事を1ヶ月テスト発信して、市場の反応を見てから改善する。**
> 最初から100点を目指すと発信できない。まず出して、反応をデータにする。

## STEP1｜市場調査リサーチ

### らっこ式：ジャンル選定3要素（リサーチ前に必ず評価）

> **売れるジャンルの条件：市場の大きさ × 悩みの深さ × 悩みの緊急度**

| 要素 | 評価基準 | 合格ライン |
|------|---------|----------|
| 市場の大きさ | そのジャンルに悩む人が多いか | Xで検索して投稿数・フォロワー数を確認 |
| 悩みの深さ | 解決したくて夜も眠れないレベルか | 表層的な不便ではなく「本音の苦しみ」か |
| 悩みの緊急度 | 今すぐ解決したいか・後回しにできないか | 「いつかやろう」ではなく「今すぐ知りたい」 |

**らっこ式ジャンル別収益規模の目安**

| ジャンル | 月間売上規模 | 価格帯 | 特徴 |
|---------|------------|--------|------|
| お金・副業 | 大 | 1,000〜5,000円 | 緊急度高、競合多い |
| 健康・ダイエット | 大 | 500〜3,000円 | 需要安定、悩み深い |
| 恋愛・結婚 | 中〜大 | 1,000〜3,000円 | 感情購買、拡散しやすい |
| 子育て・教育 | 中 | 500〜2,000円 | ママ層の購買力高い |
| キャリア・転職 | 中 | 1,000〜3,000円 | 決断期に集中購買 |
| 趣味・ライフスタイル | 小〜中 | 300〜1,500円 | 競合少、ファン化しやすい |

### らっこ式：HARMの法則（インサイト分析の軸）

ターゲットの悩みをこの4軸で分類して深掘りする。

| 軸 | 意味 | 悩みの例 |
|----|------|---------|
| **H**ealth（健康） | 身体・精神・美容の悩み | 疲れが取れない／老けた気がする |
| **A**mbition（野心） | 成長・スキル・将来への不安 | このままでいいのか／AIに仕事を奪われる |
| **R**elation（人間関係） | 家族・職場・恋愛の悩み | 子どもと話が噛み合わない／夫婦の温度差 |
| **M**oney（お金） | 収入・節約・資産形成 | 収入が増えない／老後が不安 |

> **ポイント**：ターゲットの「本音の悩み」はたいてい複数の軸にまたがる。
> 例：「AI時代の主婦の悩み」= A（時代に取り残される不安）× R（子どもに何も教えられない）× M（収入がない）

### 確認する4つの条件（必ず順番に聞く）

| # | 質問 | 選択肢例 |
|---|------|---------|
| ① | 調査テーマ | AI副業／思考法／副業術 など |
| ② | ターゲット層 | 主婦／20代会社員／フリーランス／学生 など |
| ③ | 販売形態 | eBook・ノウハウ記事／テンプレート・ツール／オンライン講座 |
| ④ | 重点項目 | インサイト深掘り／競合分析重視／ブルーオーシャン発見／パワーワード重視 |

### 出力する4項目

```
1. ターゲットの深い悩み（インサイト）× 5つ
   └ HARMの法則で分類して「夜も眠れない本音の動機」を抽出

2. 競合コンテンツの分析 × 3件
   └ タイトル・価格帯・訴求ポイント・弱点

3. 市場の空白（ブルーオーシャン）× 3つ
   └ 競合がまだ触れていないニッチな需要

4. 売れるパワーワード × 10個
   └ ターゲットの感情を揺さぶる単語
```

## STEP2｜コンセプト整理

### らっこ式：三大欲求（コンセプトの根本に置く）

> **人が情報にお金を払う動機は3つだけ。コンセプトをこの欲求に接続する。**

| 欲求 | 意味 | 購買トリガー |
|------|------|------------|
| **繁殖欲求** | 子どもを守りたい・家族を守りたい | 「子どもの未来を守る」「家族が安心できる」 |
| **承認欲求** | 認められたい・尊敬されたい・自己成長 | 「周りに差をつける」「一歩先を行く親になる」 |
| **金銭欲求** | お金を増やしたい・損したくない | 「月1万円の副収入」「AI時代に稼ぐ方法」 |

> **使い方**：タイトルとキャッチコピーに最低1つの欲求を必ず組み込む。
> 複数の欲求を組み合わせると購買力が上がる（例：繁殖×承認＝「子どもに尊敬されるお母さんになる」）

### 入力する情報

```
・ターゲット：{ターゲット}
・切り口：市場の空白になっているところ
・パワーワード：{リサーチで選んだパワーワード2〜3個}
・接続する欲求：{繁殖／承認／金銭 から選ぶ}
```

### 出力する4要素

| 要素 | 内容 |
|------|------|
| タイトル案 | 5本（パワーワードを2つ以上＋三大欲求を1つ以上含める） |
| 誰のどんな悩みを解決するか | インサイトから抽出した「本音の問題」 |
| どうなれるか | 記事を読んだ後の読者の変化 |
| 独自の売り | 競合にない切り口・市場の空白 |

### タイトル作成の公式

```
【ターゲット】 + 【パワーワード①】 + 【パワーワード②】
例：「子どもに教えられるお母さんになる　AI時代の家族を守る思考法」
```

## STEP4｜本文執筆

### 執筆ルール

```
・文字数    ：2,000文字程度／章
・トーン    ：親しみやすく、少し熱量のある口調
・構成      ：PREP法（結論→理由→具体例→結論）
・具体例    ：失敗談・成功例を必ず1つずつ入れる
・読者視点  ：「あなたは〇〇ではありませんか？」で引き込む
```

### PREP法の構造

```
【結論】  最初に「何を伝えたいか」をはっきり言う
   ↓
【理由】  「なぜそう言えるのか」根拠を説明する
   ↓
【具体例】 失敗談・成功例・データで実感させる
   ↓
【結論】  最初の結論を繰り返し、次章へ誘導する
```

### 執筆順序

```
① はじめに（フック・本書の目的）
② 第1章〜第6章（各2,000文字）
③ おわりに（復習・継続の呼びかけ・最後の一押し）
```

### 各章の書き出しパターン

| 章 | 冒頭パターン |
|----|------------|
| はじめに | 具体的な場面描写から始める（「ある夜のことです」） |
| 第1章 | 「こんな場面に心当たりはありませんか？」 |
| 第2章 | 「あなたの周りに、こんな人はいませんか？」 |
| 第3章 | 「〇〇と聞いて、こう思いませんでしたか？」 |
| 第4章 | 「でも、これって本当に効果があるの？」 |
| 第5章 | 「わかった。でも具体的に何から始めればいいの？」 |
| 第6章 | 「ここまで読んでくれて、ありがとうございます」 |
| おわりに | 読み始めたときの読者の気持ちを代弁する |

```
└── {記事タイトル}.docx    ← 完成ファイル
```

### スクリプトの基本構造

```javascript
import { Document, Packer, Paragraph, TextRun, HeadingLevel, AlignmentType } from "docx";
import { writeFileSync } from "fs";

const doc = new Document({
  styles: {
    default: {
      document: { run: { font: "游明朝", size: 24 } },
    },
  },
  sections: [{
    children: [
      // タイトル
      new Paragraph({
        text: "{タイトル}",
        heading: HeadingLevel.TITLE,
        alignment: AlignmentType.CENTER,
      }),
      // 見出し
      new Paragraph({ text: "第1章｜...", heading: HeadingLevel.HEADING_1 }),
      // 小見出し
      new Paragraph({ text: "結論：...", heading: HeadingLevel.HEADING_2 }),
      // 本文
      new Paragraph({ text: "本文テキスト..." }),
      // 強調テキスト
      new Paragraph({
        children: [new TextRun({ text: "強調したい文章", bold: true, size: 28 })],
        alignment: AlignmentType.CENTER,
      }),
    ],
  }],
});

const buffer = await Packer.toBuffer(doc);
writeFileSync("C:/Users/owner/dev/note/{ファイル名}.docx", buffer);
console.log("完成しました！");
```

## STEP7｜SNS告知文

### らっこ式：X→note順序の原則

> **「Xで読者を集めてからnoteを売る」がらっこ式の基本戦略。**
> note単体で売ろうとしない。Xで価値提供→信頼獲得→noteへ誘導の流れを作る。

```
❌ NG：noteを公開してからXで宣伝する（読者ゼロからスタート）
✅ OK：Xで発信して読者を増やしてからnoteを出す（温まった読者に届ける）
```

**理想の導線：**
```
Xで日常的に価値提供（知識・体験・共感）
         ↓
フォロワーが「この人の有料コンテンツも読みたい」と思う状態を作る
         ↓
noteを公開して告知（すでに信頼がある状態）
         ↓
初速で購入・レビューが入る
```

> **Xフォロワー目安**：フォロワー500〜1,000人いれば初速で10〜20件購入が見込める。
> フォロワーゼロから始める場合は、まずXで30日間発信してから有料記事を出す。

### X（ツイート）パターン別使い分け

| パターン | 使うタイミング | 狙い |
|---------|--------------|------|
| 共感型 | 公開初日 | 「私のことだ」で拡散 |
| 問題提起型 | 3日後 | 危機感を煽って購買促進 |
| 数字・実績型 | レビューが集まってから | 信頼性で背中を押す |
| スレッド型 | 土曜の朝 | 情報提供で拡散狙い |
